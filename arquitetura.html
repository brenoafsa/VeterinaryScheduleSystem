Arquitetura do Sistema

O sistema segue a arquitetura MVC (Model-View-Controller), que permite uma clara separação entre a interface, a lógica de negócio e o gerenciamento dos dados. O Modelo (Model) representa as classes e estruturas de dados, como consultas, tutores e pets. A Visão (View) é responsável pela interface com o usuário, enquanto o Controlador (Controller) gerencia as interações entre a interface e os dados.
A implementação foi modularizada utilizando arquivos de cabeçalho (.h) e arquivos de implementação (.cpp). As classes foram organizadas conforme os módulos principais: cadastro de clientes, agendamentos e consultas. O uso dessa estrutura melhora a manutenção e expansibilidade do sistema.
A interface gráfica foi criada com o Qt Designer, que gera arquivos .ui contendo descrições visuais de botões, campos de texto e demais componentes. Esses arquivos são integrados ao código C++ para criar a interatividade necessária no sistema.

Tecnologias Utilizadas

O sistema foi desenvolvido utilizando as seguintes tecnologias:

  • Linguagem de Programação: C++
  • Framework de Interface: Qt Framework
  • Armazenamento de Dados: JSON
  • Ferramenta de Interface: Qt Designer

O Qt Framework é uma poderosa biblioteca para desenvolvimento de interfaces gráficas, oferecendo suporte para a interação entre elementos visuais e a lógica do sistema. Além disso, ele permite manipulação de arquivos JSON, criação de janelas e interatividade de elementos na tela.
Os dados são armazenados localmente utilizando arquivos JSON, escolhidos por serem leves, fáceis de manipular e não exigirem um banco de dados relacional. Isso permite maior simplicidade e portabilidade ao sistema, garantindo que informações como consultas, tutores e pets sejam acessadas rapidamente.
