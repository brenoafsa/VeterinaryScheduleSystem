Modelagem Orientada a Objetos

O projeto segue os princípios da orientação a objetos, garantindo modularidade, reutilização de código e manutenção facilitada. Os principais conceitos aplicados são:

  • Encapsulamento: As classes encapsulam dados e comportamentos, restringindo o acesso direto a informações sensíveis. A comunicação entre classes ocorre por meio de métodos públicos, garantindo maior segurança e organização do código.

  • Herança: As classes de interface gráfica herdam de QDialog para criar janelas modais, permitindo a reutilização de código e a padronização da interface.

  • Polimorfismo: Métodos virtuais são utilizados para permitir a sobrescrita de funcionalidades em classes derivadas, possibilitando maior flexibilidade na implementação de diferentes comportamentos.

  • Composição: Diversas classes fazem uso da composição, integrando objetos como QPushButton, QTreeWidget e outros componentes gráficos para formar interfaces mais dinâmicas e interativas.

  • Agregação e Associação: A modelagem do sistema permite que objetos de diferentes classes se relacionem, como um tutor que pode possuir múltiplos pets, reforçando a coerência dos dados.
